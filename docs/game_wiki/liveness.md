# 活跃任务 (Liveness)

> 7 任务之一。**V1 已实现,真机回归待跑**(2026-06-27 dryrun_liveness 失败,模板不匹配)。

---

## 1. 任务名称

活跃任务(每日任务 / Liveness)

## 2. 任务作用

- 完成每日活跃任务(通关 X 次副本/完成 X 次签到/...)
- 累计活跃度换奖励(铜币/钻石/体力/忍者碎片/...)
- 每日 0 点刷新

## 3. 入口位置

### 入口 A:主页 → 奖励中心 → 活跃卡片
```
[主页]
  ↓ 点击右下"奖励"礼物盒 (1820, 555)
[奖励中心页]
  ↓ 点击"活跃"卡片(左侧垂直 3 tab 第二项)
[活跃页]
  ↓ 完成任务(每个活跃任务有"立即前往"按钮)
  ↓ 点击"领取"按钮(已完成的任务)
[活跃页]
  ↓
  ↓ 关闭奖励中心
[主页]
```

### 入口 B:主页 → 活动页 → 左侧菜单 → 活跃
```
[主页]
  ↓ 点击右上"活动"图标 (1820, 85)
[活动页]
  ↓ 左侧菜单(可能含"活跃" tab,需 swipe 找)
[活跃 tab]
  ↓ 完成 / 领取
[活动页]
  ↓
[主页]
```

> **推荐用入口 A**(更直接);**入口 B 待确认**。

## 4. 页面截图说明

### 4.1 主页(奖励入口)

- **位置**: 右侧中部 `(~1820, 555)` 橙色礼物盒
- **颜色**: 橙底 + 金色蝴蝶结 + "奖励"白字 + 红点
- **检测**: 礼物盒轮廓 + 蝴蝶结 + "奖励"字

### 4.2 奖励中心页

- **顶部**: "奖励中心"标题 + 右上 X `(1860, 60)` 关闭
- **左侧**: 垂直 3 tab
  - 每日任务
  - 奖励中心(默认选中)
  - 日历
- **中部**: 卡片网格(签到 / 活跃 / 商城 / 限时 / ...)

### 4.3 活跃页

- **顶部**: "活跃"标题 + 关闭 X
- **中部**: 任务列表
  - 每行: 任务名 + 进度(0/3) + 奖励 + "立即前往" / "领取" 按钮
  - 已完成的任务: 灰底 + "领取" 按钮亮
  - 未完成: 进度 + "立即前往" 按钮
- **底部**: 当日总活跃度 + 累计奖励

### 4.4 任务完成结果弹窗

- **中央**: "任务完成" + 奖励图标
- **底部**: "确定"按钮

## 7. 任务完整流程

### 入口 A(推荐)

```
[主页]
  ↓ find_award_button 节点(ROI (1760, 460, 200, 180), 礼物盒模板)
[检测到礼物盒 @ (1820, 555), conf=0.99]
  ↓ tap (1820, 555) + tap_offset_y=-37 → 实际 tap (1820, 518)
[奖励中心页]
  ↓
  ↓ find_liveness_card 节点(找"活跃"卡片)
[检测到"活跃"卡片]
  ↓ tap 卡片中央
[活跃页]
  ↓
  ↓ 循环: 找"领取"按钮 → tap → 弹窗 → 关弹窗
  ↓ 直到所有任务已领取
[活跃页(无未领取任务)]
  ↓
  ↓ 关闭活跃页 → 关闭奖励中心
[主页]
  ↓
  ↓ verify_done 节点
[SUCCESS]
```

## 8. 成功条件

- **主页判定通过**
- **活跃页所有可领取任务已领取**
- **没有异常弹窗**

## 9. 失败情况

| 情况 | 原因 | 解决 |
|---|---|---|
| 入口礼物盒 conf<0.75 | 模板视觉差异大 | 多候选模板 + ROI 扩大 |
| 活跃卡片找不到 | 卡片位置可能变 | OCR 兜底找"活跃"文字 |
| 活跃任务全部已完成 | 之前已领完 | SUCCESS 退出 |
| "领取"按钮被遮挡 | 弹窗阻挡 | dismiss_x 关闭后重试 |
| 活跃页 X 位置错 | 误以为在右上角 | 实际在左上 |
| 教学弹窗挡 | 第一次进活跃页 | dismiss_x 关闭 |
| 网络异常 | 领取时断线 | 重新进活跃页 |
| "立即前往"误点 | 跳转到具体任务 | 误点;按"返回"回活跃页 |

## 10. 恢复方式

```
[检测到非主页元素 / 任务卡住]
  ↓
[1. 关闭弹窗]
  - dismiss_x 候选模板扫描 3 次
  - 活跃页 X (左上)
  - 奖励中心 X (1860, 60)
  ↓
[2. 重试任务]
  - find_award_button → tap
  - find_liveness_card → tap
  - find_claim_button → tap (循环)
  ↓
[3. 验证主页]
  - 检测资源栏 + 6 圆盘按钮
  ↓
[4. 主页 → SUCCESS]
  - 3 次循环后仍未到主页 → 报 FAIL
```
