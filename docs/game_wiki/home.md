# 主页 (Home)

> 所有任务的起点和终点。**自动化所有任务的第一段都是 ensure_home** — 把画面拉回到这个页面。

---

## 1. 任务名称

主页(主城 / 战斗主页 / 出生点)

## 2. 任务作用

- 玩家在游戏中的**唯一主城**;所有任务(邮件/签到/活跃/组织/招募/活动/商店/战斗副本/...)都从主页出发
- 显示**角色状态**(头像/等级/战力/体力/铜币/声望/钻石)
- 显示**功能入口**(邮件/奖励/活动/商城/招募/...)
- 显示**顶部时间倒计时**(功夫季/幻之试炼/送S忍等限时活动)
- 显示**战斗副本入口**(中间决斗场/组织/小队突袭/丰饶之间/排行)

## 3. 入口位置

- 任意页面 → **底部"返回"按钮** `(110, 990)` → 主页
- 任意页面 → **顶部 X 关闭按钮** → 主页
- 战斗结算页 → 退出战斗 → 主页
- 异常掉线 → 重连 → 主页
- **主页判定**:画面顶部有 `角色头像 + 等级 + 战斗力 + 资源栏(体力/铜币/声望/钻石)`,中央有 **决斗场 / 组织 / 排行 / 决战场** 4 个横向建筑,底部有 6 个 **忍者 / 天赋 / 装备 / 通灵 / 秘卷 / 装扮** 圆盘按钮

## 4. 页面截图说明

**游戏整体布局是 1920×1080 横屏**(以 MuMu 模拟器 127.0.0.1:16384 实际采集为准)。

### 顶部资源栏 (y=0..80)
- 左侧:角色头像 + 等级 + 战斗力 + V几
- 中部:体力 `(13/200)` + 铜币 + 声望/钻石
- 右侧:功夫季(天数倒计时,绿字 + 红球) + 活动图标

### 顶部公告区 (y=80..200)
- 左侧:跨服要塞战倒计时(`8:10:04` 黄字 + 战区分组倒计时)
- 右侧:幻之试炼 5 / 送 S 忍 / 商城(垂直 4 个小图标,带红点)

### 邮件/奖励侧栏 (x=0..150, y=250..550)
- **黄色信封** `(30, 380)` + 红 2 角标(邮件未读)
- **橙色礼物盒** `(~1820, 555)`(奖励入口)
- **摄像机图标** `(75, 200)` 附近(录像/回忆入口)

### 中央主城 (x=400..1500, y=200..800)
- **左侧建筑**:积分赛/组织(带红点)
- **中央建筑**:决斗场 / 决战场(带红点提示限时活动)
- **右侧建筑**:忍者 / 小队突袭(带红点)
- **中下**:丰饶之间 / 排行 / 会所
- **中央底部**:右下角**忍界指引卷轴图标** `(1580, 940)` + 底部黄色云朵(装饰,图标本身一部分) + **冒险白底竖牌** `(1810, 1020)`(白底+黑色"冒 险"字+红色装饰边框+绳子缠绕+底部黄色云朵)

### 底部导航栏 (y=950..1080)
- **6 个圆盘按钮**(横向并排):
  - `(120, 1000)` 忍者
  - `(280, 1000)` 天赋
  - `(450, 1000)` 装备
  - `(650, 1000)` 通灵
  - `(850, 1000)` 秘卷
  - `(1050, 1000)` 装扮
- **底部状态条**:邮件小图标/世界/组织/聊天框(左侧 3 个喇叭/摄像头/头像)

### 背景
- 漩涡亮绿(主城背景,新账号"漩涡鸣人 38级 V8 不负闪光之名")
- 旧账号鼬 147级时是暗灰背景
- 背景**用户可主动选**;同按钮在不同账号/不同背景下视觉差异大,模板必须 fallback

## 6. 页面变化元素

| 元素 | 变化原因 | 模板策略 |
|---|---|---|
| 顶部倒计时(功夫季天数) | 每天 -1 | 不作为模板,只作为状态提示 |
| 跨服要塞战倒计时 | 比赛时间 | 不作为模板 |
| 顶部活动横幅(限时/广告) | 每周更新 | 不作为模板 |
| 背景图 | 用户可选 | 模板必须多 fallback |
| 主城建筑红点 | 任务状态/活动推送 | 变化频繁,不能作为模板 |
| 中部"挑战获得大量经验,铜币和声望"教学弹窗 | 教学任务 | 教学弹窗,会挡其他按钮 |
| 世界/组织/战区聊天框 | 玩家聊天内容 | 不作为模板,只作为存在判定 |
| 左下角邮件小图标 | 邮件未读 | 数值可变,形状固定 |

## 7. 任务完整流程

```
[任意任务子页]
  ↓ 点击关闭 X
[关闭弹窗 / 弹窗消失]
  ↓
[仍可能在该子页]
  ↓ 点击底部"返回"按钮 (110, 990)
[主页]
  ↓ ensure_home 完成
```

**典型 ensure_home 节点设计**:
1. **检测主页模板**(资源栏 + 6 圆盘按钮 + 顶部头像)
2. **如果在主页 → 跳过 → SUCCESS**
3. **如果不在主页 → 关闭所有弹窗**(顶部 X + dismiss_x 候选)
4. **按"返回"按钮 `(110, 990)` 一次**
5. **再次检测主页**
6. **重复 3-5 步,最多 3 次**
7. **如果还不在主页 → 强行重启游戏或报 FAIL**

## 8. 成功条件

- 画面包含:**角色头像 + 资源栏(体力/铜币) + 底部 6 圆盘按钮(忍者/天赋/装备/通灵/秘卷/装扮)**
- **没有任何全屏弹窗**遮挡(教学弹窗/活动弹窗/奖励弹窗)
- **没有任何非主页的内层页面**(邮件页/活动页/奖励中心页/...)

## 9. 失败情况

| 情况 | 原因 | 解决 |
|---|---|---|
| 教学弹窗遮挡 | 教学剧情弹窗(中央"挑战获得大量经验...") | 点中央确认;或 X 关闭 |
| 活动全屏弹窗 | "X 天后开启新活动" / 登录奖励弹窗 | 点 X 关闭 |
| 战区聊天弹窗 | 玩家发言触发(屏幕中央大字) | 自动消失或 X 关闭 |
| 段位奖励弹窗 | 段位赛结束后 | "OK" 按钮确认 |
| 跨服要塞战弹窗 | 比赛开始前倒计时 | 等待或 X 关闭 |
| 商店弹窗 | 误点"商城" | (1820, 85) 活动或 X |
| 网络异常 | 服务器断开 | "重连"按钮(可能变化) |
| 资源加载中 | 服务器慢 | 等待(可加重试) |
| 维护公告 | 服务器维护中 | 不支持自动化 |

## 10. 恢复方式

| 弹窗类型 | 关闭方式 |
|---|---|
| 教学剧情弹窗(中央) | 点中央空白(不会自动关) |
| 顶部全屏活动弹窗 | 顶部 X 关闭 `(1860, 60)` |
| 段位奖励/通知弹窗 | "OK" 按钮(中央偏下) |
| 跨服要塞战倒计时 | 等待倒计时结束或 X 关闭 |
| 战区聊天大字 | 自动消失(几秒) |
| 误入副本页 | `(110, 990)` 返回 |
| 误入积分赛页 | `(110, 990)` 返回 |
| 误入冒险副本页 | `(110, 990)` 返回 |
| 误入一乐外卖 | `(110, 990)` 返回 |
| 误入商城 | `(110, 990)` 返回 |
| 网络异常 | "重连"按钮(位置不固定) |
| 维护中 | 无法恢复,任务 FAIL |

## 11. 自动化建议

| 场景 | 推荐技术 | 原因 |
|---|---|---|
| 主页判定 | **多模板匹配**(资源栏 + 6 圆盘按钮 + 头像) | 主页最稳,多 fallback |
| ensure_home | **多模板 + dismiss_x + 返回按钮** | 关闭弹窗 → 按返回 → 验证 |
| 邮件信封 | **多尺度模板匹配**(scale=0.85..1.15) | 信封较小,需多尺度 |
| 奖励礼物盒 | **多尺度模板匹配** | 礼物盒小 |
| 活动图标 | **多尺度模板匹配** | 顶部图标小 |
| 忍界指引卷轴 | **多尺度模板匹配** + tap_offset_y=-0.25 | 按钮本体只占视觉上半部分 |
| 冒险白底竖牌 | **多尺度模板匹配** + tap_offset_y=-0.25 | 同上 |
| 顶部 X 关闭 | **多模板候选**(shared/x*.png) | X 在左上角(不是右上角!) |
| 背景识别 | **不用背景** | 背景用户可换,不能依赖 |
| 顶部倒计时 | **不用倒计时** | 每天 -1,变 |
| 战区聊天内容 | **不用聊天内容** | 内容是玩家输入,变 |
