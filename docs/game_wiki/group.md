# 组织 / 祈福 (Group / Group Signin)

> 7 任务之一。**V1 §1.2.2 已重构入口段,真机跑通**(2026-06-26 旧账号 conf=0.99 + 1.00 双节点命中)。**入口是"忍界指引→组织 tab→即刻前往"**(2026-06-29 user 确认)。

---

## 1. 任务名称

组织祈福(group_signin / 组织铜币祈愿 / 组织追击)

## 2. 任务作用

- 完成"加入 1 个组织"日常任务
- 完成"发起 1 次祈愿"日常任务(铜币祈愿)
- 完成"组织商店购买 1 次"日常任务
- 4 个子任务:try_copper_pray(发起铜币祈愿)+ confirm_copper_pray(确认)+ try_pursuit_entry(进追击界面)+ try_pursuit(发起追击)

## 3. 入口位置

### 入口 A(新,2026-06-29 user 确认,推荐)

```
[主页]
  ↓ 点击主页右下"忍界指引"卷轴图标 (1580, 940)
[忍界指引页]
  ↓ 点击左侧"组织" tab (默认可见第 2 行 ~(110, 320))
[组织 tab 页]
  - 顶部标题:"组织"
  - 中间海报:"加入组织,寻找和你志同道合的人吧!" + 底部"即刻前往"按钮 (1325, 815)
  - 右侧任务:加入 1 个组织(已完成)/ 发起 1 次祈愿(已完成)/ 组织商店购买 1 次(已完成)
  ↓ 点击中间"即刻前往"按钮 (1325, 815)
[组织页 / 加入组织流程 / 已加组织]
  ↓ check_no_group 节点判断是否已加组织
[已加组织]
  ↓
  ↓ 4 子链路
  ├─ try_copper_pray(发起铜币祈愿)
  ├─ confirm_copper_pray(确认祈愿)
  ├─ try_pursuit_entry(进追击界面)
  └─ try_pursuit(发起追击)
[回到主页]
```

### 入口 B(旧,V1 §1.2.2 校准过,真机跑通)

```
[主页]
  ↓ 点击右下"奖励"礼物盒 (1820, 555)
[奖励中心页]
  ↓ 点击"组织祈福"卡片
[组织祈福页]
```

> **新文档以入口 A 为准**;**入口 B 作为 fallback 候选**(V1.2 §1.2.2 校准数据保留在 `group_signin.md` 历史版本)。

## 8. 成功条件

- **主页判定通过**
- **"加入 1 个组织"任务已完成**(新账号下已默认完成)
- **"发起 1 次祈愿"任务已完成**
- **"组织商店购买 1 次"任务已完成**
- **4 子链路全跑通**(try_copper_pray / confirm_copper_pray / try_pursuit_entry / try_pursuit)

## 9. 失败情况

| 情况 | 原因 | 解决 |
|---|---|---|
| **未加组织**(check_no_group 短路) | 玩家没加组织 | 提示用户加组织后跑 |
| 忍界指引卷轴找不到 | 模板视觉差异大 | 多候选模板 + OCR 兜底 |
| 左侧菜单"组织" tab 找不到 | 菜单可滑动 | 滑回顶部找"组织" |
| "即刻前往" 按钮位置错 | 误以为在右上角 | 实际在中间 (1325, 815) |
| 加入组织流程卡住 | 选组织/申请需人工 | 任务 FAIL |
| 4 子链路阻塞 | check_no_group 短路 | 同上 |
| 教学弹窗挡 | 第一次进组织页 | dismiss_x 关闭 |
| 弹窗阻挡 | 段位奖励/活动 | dismiss_x 关闭 |
| 网络异常 | 祈愿/追击时断线 | 重新进组织页 |

## 10. 恢复方式

```
[检测到非主页元素 / 任务卡住]
  ↓
[1. 关闭弹窗]
  - dismiss_x 候选模板扫描 3 次
  - 忍界指引页 X (1860, 60)
  - 组织页 X (左上)
  ↓
[2. 重试任务]
  - 4 子链路重试 3 次
  ↓
[3. 验证主页]
  - 检测资源栏 + 6 圆盘按钮
  ↓
[4. 主页 → SUCCESS]
  - 3 次循环后仍未到主页 → 报 FAIL
```
