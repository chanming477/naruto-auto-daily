# 通用 UI 元素 (Common UI)

> 所有页面都可能出现的基础 UI 元素。**RecoveryManager 最重要的知识来源** — 几乎所有"卡住恢复"都靠识别这些元素。

---

## 1. 任务名称

通用 UI 元素清单

## 2. 任务作用

- 维护跨页面通用的 UI 元素清单(关闭/确认/取消/返回/主页/加载/网络异常/更新/公告/奖励弹窗)
- RecoveryManager 用这些元素做"卡住恢复"
- 模板采集优先级高(几乎每个任务都要用)

## 3. 入口位置

> 通用 UI 元素**没有固定入口**,会在任意页面出现。

---

## 4. 页面截图说明

> 通用 UI 元素分散在所有页面,以下按"出现频率从高到低"排序。

### 4.1 关闭按钮 (X) — ⭐ 最重要

**位置**: 弹窗的**左上角**(不是右上角!) — 2026-06-27 用户明确澄清

| 类型 | 位置 | 颜色 | 备注 |
|---|---|---|---|
| 全屏页面 X | 屏幕左上角 `(80, 60)` 附近 | 红 X | 关闭整个全屏页(忍界指引/活动页/...) |
| 子弹窗 X | 弹窗左上角 | 红 X | 关闭单个弹窗 |
| 邮件页 X | `(1760, 187)` | 红 X | 关闭邮件 |
| 奖励中心 X | `(1860, 60)` 附近 | 红 X | 关闭奖励中心 |
| 商店页 X | 左上角 `(80, 60)` 附近 | 红 X | 关闭商店 |
| 商城页 X | 左上角 | 红 X | 关闭商城 |
| 任务页 X | 顶部居中 | 红 X | 关闭任务页 |
| 忍者详情 X | 顶部居中 | 红 X | 关闭忍者详情 |
| 战斗结算 X | 不一定 | 可能是"再战" / "退出" | 不强制 |

> **关键**: X 永远在弹窗**左上角**。早期版本误以为在右上角,导致点错位置。

### 4.2 确认/取消按钮 (OK / Cancel)

| 按钮 | 位置 | 颜色 | 备注 |
|---|---|---|---|
| 段位赛"领取" | 段位奖励弹窗中央 `(960, 750)` | 蓝底 | 段位赛结束后 |
| "再次挑战" | 战斗结算页中央 | 蓝底 | 战斗完成后 |
| "退出" | 战斗结算页右侧 | 灰底 | 退出战斗回主页 |
| "确认" | 通用弹窗中央偏下 | 蓝/橙底 | 各种确认 |
| "取消" | 通用弹窗中央偏左 | 灰底 | 各种取消 |
| "立即前往" | 活动海报底部 `(1325, 815)` | 橙底 | 进任务界面 |
| "前往" | 任务列表右侧 | 橙底 | 跳转到对应任务 |

### 4.3 返回/主页按钮

| 按钮 | 位置 | 颜色 | 备注 |
|---|---|---|---|
| **返回主页**(底部) | `(110, 990)` | 蓝/绿底 | **多页面通用的返回按钮** — 误点(535, 510)等位置也是返回 |
| **忍者页底部"忍者"** | `(100, 1000)` | 蓝底 | 进忍者页 |
| **忍者页 X** | `(1820, 55)` | 红 X | 关闭忍者页 |
| **主页按钮**(弹窗内) | 弹窗底部 | 蓝底 | 各种弹窗"回主页" |

### 4.4 加载动画

- 旋转的漩涡 / 苦无 / 树叶图标
- **位置**: 通常在屏幕中央
- **颜色**: 蓝/白
- **检测**: 模板特征明显,但**形状不变**(是稳定的)

### 4.5 网络异常

| 状态 | 表现 | 解决 |
|---|---|---|
| 断线 | "网络连接断开" + "重连"按钮 | 点"重连" |
| 超时 | 加载圈转不停 → 5s 后弹窗 | 等 5s 后看弹窗 |
| 服务器维护 | "服务器维护中" + 时间 | 无法恢复,任务 FAIL |

### 4.6 更新提示

- **触发**: 游戏版本更新
- **表现**: 全屏弹窗"检测到新版本,请更新" + "立即更新"按钮
- **解决**: 只能点"立即更新"或"取消"退出游戏(后者让任务 FAIL)
- **频率**: 每月 1-2 次

### 4.7 公告

- **触发**: 登录时
- **表现**: "本周更新公告"全屏 + 滚动文本 + "我知道了"按钮(右下)
- **解决**: 点"我知道了"或拖到底部才能点
- **频率**: 每周一次(周三维护后)

### 4.8 奖励弹窗

- **触发**: 完成任务/签到/升级/活动
- **表现**: 中央全屏 + 奖励图标 + "确定"按钮
- **解决**: 点"确定"或点中央
- **可能变**: 奖励内容每天变,**按钮位置和颜色固定**

### 4.9 充值弹窗

- **触发**: 误点商店/活动推送
- **表现**: "首充礼包" + "购买"按钮
- **解决**: X 关闭(左上角)
- **频率**: 误触

### 4.10 战区聊天

- **触发**: 玩家在战区/世界/组织频道发言
- **表现**: 屏幕中央滚动大字 + 头像 + 文本
- **解决**: 等几秒自动消失
- **影响**: 临时遮挡中央按钮

## 5. 页面固定元素

| 元素 | 位置 | 模板价值 |
|---|---|---|
| **全屏页面 X**(左上角) | `(80, 60)` 附近 | ⭐⭐⭐ 通用关闭,几乎所有页都有 |
| **返回主页按钮** | `(110, 990)` | ⭐⭐⭐ 通用返回 |
| **通用"确定"按钮**(蓝底) | 弹窗中央偏下 | ⭐⭐ 通用确认 |
| **通用"取消"按钮**(灰底) | 弹窗中央偏左 | ⭐ 通用取消 |
| **加载动画** | 屏幕中央 | ⭐ 等待判定 |
| **网络重连按钮** | 不固定(弹窗中央) | ⭐ 异常恢复 |
| **更新"立即更新"按钮** | 弹窗中央 | 普通 |
| **公告"我知道了"按钮** | 弹窗右下 | ⭐ 公告关闭 |
| **奖励"确定"按钮** | 弹窗中央 | ⭐ 奖励关闭 |
| **充值"购买"按钮** | 弹窗中央 | 普通 |

## 6. 页面变化元素

| 元素 | 变化原因 | 模板策略 |
|---|---|---|
| 公告文本 | 每周更新 | 不能作为模板 |
| 更新版本号 | 每月更新 | 不能作为模板 |
| 充值礼包内容 | 节假日/活动 | 不能作为模板 |
| 奖励内容 | 任务奖励随机 | 不能作为模板 |
| 战区聊天文本 | 玩家输入 | 不能作为模板 |

## 7. 任务完整流程

```
[任意弹窗/全屏页]
  ↓
[识别弹窗类型]
  ├─ 教学剧情 → 点中央继续
  ├─ 全屏活动 → 点左上 X 关闭 (80, 60)
  ├─ 段位奖励 → 点"领取" (960, 750)
  ├─ 网络异常 → 点"重连"
  ├─ 更新提示 → 点"立即更新"或"取消"退出
  ├─ 公告 → 点"我知道了"或拖到底
  ├─ 奖励 → 点"确定"
  ├─ 充值 → 点 X 关闭
  ├─ 战区聊天 → 等待
  └─ 误入子页 → 点"返回主页" (110, 990)
  ↓
[主页或下一级页面]
```

## 8. 成功条件

- 弹窗/全屏页成功关闭
- 没有全屏遮挡
- 主页判定通过(详见 [home.md](./home.md) §8)

## 9. 失败情况

| 情况 | 原因 | 解决 |
|---|---|---|
| X 位置识别错误 | 误以为在右上角 | 用 dismiss_x 候选模板(shared/x*.png),多候选 |
| 返回按钮失灵 | 误入战斗/战斗结算页 | 战斗结算的"退出"按钮位置不同 |
| 网络重连失败 | 服务器挂 | 多次重试,5 次后 FAIL |
| 公告无法关闭 | "我知道了"按钮在右下但被遮挡 | 拖到底再点 |
| 更新提示无法跳过 | 版本强制更新 | 任务 FAIL,需人工 |
| 维护中 | 服务器维护 | 任务 FAIL,需人工 |
| 战区聊天长期不消失 | 服务器卡 | 等 30s 后强制关闭 |
| 充值弹窗无法关闭 | 强制充值 | 极少见,需人工 |

## 10. 恢复方式

### 10.1 标准恢复流程

```
[检测到非主页元素]
  ↓
[1. 关闭所有弹窗]
  - dismiss_x 候选模板扫描 3 次
  - 顶部 X (80, 60) / 邮件 X (1760, 187) / 奖励中心 X (1860, 60) 等
  ↓
[2. 按"返回主页" (110, 990)]
  - 最多 3 次
  ↓
[3. 验证主页]
  - 检测资源栏 + 6 圆盘按钮
  ↓
[4. 主页 → SUCCESS]
  - 非主页 → 重复 1-3
  ↓
[5. 3 次循环后仍未到主页]
  - 报 FAIL,需要人工介入
```

### 10.2 战斗页特殊恢复

- 战斗结算页:点"退出"按钮(右侧,灰底)
- 战斗中:无法直接退出,需等战斗结束
- 战斗断线:自动重连 → 进战斗结算

### 10.3 弹窗特化恢复

| 弹窗 | 关闭方式 |
|---|---|
| 教学剧情(中央) | 点中央空白(不自动关) |
| 顶部全屏活动(中间区) | 顶部 X (80, 60) |
| 段位奖励(右下) | (960, 750) |
| 网络异常(中央) | "重连"按钮(位置不固定) |
| 公告(右下) | "我知道了"(右下角) |
| 奖励(中央) | "确定"(中央) |
| 充值(中央) | X 关闭(左上角) |

## 11. 自动化建议

| 场景 | 推荐技术 | 原因 |
|---|---|---|
| 关闭 X 按钮 | **多候选模板 + dismiss_x** | 不同页面 X 位置略不同 |
| 返回主页按钮 | **多尺度模板匹配 + tap_offset_y** | 按钮本体占视觉上半 |
| 确认/取消按钮 | **多候选模板** | 颜色和文字固定(蓝底白字) |
| 加载动画 | **模板匹配**(稳定) | 形状不变 |
| 网络重连 | **OCR 兜底** | 文字"重连" + 位置 |
| 公告"我知道了" | **OCR 兜底** | 文字"我知道了" |
| 奖励"确定" | **多候选模板** | 蓝底白字,稳定 |
| 战区聊天 | **不用模板**(等自动消失) | 文本内容是玩家输入 |
| 充值弹窗 | **dismiss_x 关闭** | 强制充值,极少见 |
| **背景识别** | **不用背景** | 背景用户可换,不能依赖 |
| **顶部 X (80, 60)** | **多尺度模板 + scale=0.85..1.15** | X 是固定元素,需多尺度 |
