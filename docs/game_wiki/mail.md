# 邮件领取 (Mail)

> 7 任务之一。**V1 已实现,真机回归通过**(2026-06-27 新账号"漩涡鸣人 38级 不负闪光之名"conf=0.617 / 1.000 / 0.632 通过)。

---

## 1. 任务名称

邮件领取

## 2. 任务作用

- 领取系统邮件中的奖励(铜币/钻石/体力/忍者碎片/...)
- 系统邮件会定期推送:登录奖励/活动奖励/补偿邮件/...

## 3. 入口位置

```
[主页]
  ↓ 点击主页右上黄色信封
[邮件列表页]
  ↓ 点击"一键领取"按钮
[领取结果弹窗]
  ↓ 点击"确定"
[邮件列表页(已领取)]
  ↓ 点击右上 X 关闭
[主页]
```

## 4. 页面截图说明

### 4.1 主页(邮件入口)

- **位置**: 左侧中部 `(30, 380)` 黄色信封图标
- **颜色**: 黄底 + 红 2 角标(未读)
- **检测**: 信封轮廓 + 红色数字"2" / "1" 等

### 4.2 邮件列表页

- **顶部**: "邮件"标题 + 右上 X `(1760, 187)` 关闭
- **左侧**: 邮件分类 tab(全部 / 系统 / 战斗 / ...)
- **中部**: 邮件列表(每行:标题 + 摘要 + 时间 + 附件)
- **底部**: "一键领取"按钮 `(670, 932)` + 邮件数量显示
- **整体**: 占 80% 屏宽 + 80% 屏高(中央弹层)

### 4.3 领取结果弹窗

- **中央**: "领取成功!" + 奖励图标列表(铜币 + 钻石 + 体力 + 碎片)
- **底部**: "确定"按钮(中央偏下,蓝底)
- **颜色**: 金色背景 + 白色文字

## 5. 页面固定元素

| 元素 | 位置 | 模板价值 |
|---|---|---|
| **主页邮件信封**(黄底 + 红 2 角) | `(30, 380)` | ⭐⭐⭐ 入口 |
| **邮件页 X** | `(1760, 187)` | ⭐⭐⭐ 关闭 |
| **邮件页"一键领取"按钮** | `(670, 932)` | ⭐⭐⭐ 领取 |
| **领取结果"确定"** | 弹窗中央 | ⭐⭐ 关闭弹窗 |
| 邮件分类 tab(左侧) | `(0, 200, 200, 800)` | 普通(分类可能不固定) |
| 邮件列表 | 全屏中上部 | 变(内容变) |
| 邮件数量显示 | "一键领取"上方 | 变(数量变) |

## 6. 页面变化元素

| 元素 | 变化原因 | 模板策略 |
|---|---|---|
| 邮件列表内容 | 系统推送 | 不作为模板 |
| 邮件数量 | 实时变化 | 数字变,不能作为模板 |
| 邮件分类 tab | 可能新增/移除 | 多候选模板 |
| 邮件图标(每行左侧) | 邮件类型变 | 不作为模板 |
| 红色角标数字 | 未读数 | 不作为模板 |
| 背景 | 主题色 | 不作为模板 |

## 8. 成功条件

- **主页判定通过**(资源栏 + 6 圆盘按钮)
- **邮件 X 已点击**(已回到主页)
- **"一键领取"已完成**(邮件列表已无未领取邮件)
- **没有异常弹窗**

## 9. 失败情况

| 情况 | 原因 | 解决 |
|---|---|---|
| 邮件入口没找到 | 模板 conf 不达 0.75 | 多候选模板 + ROI 扩大 |
| 邮件 X 位置错 | 误以为在右上角 | **X 在 (1760, 187) 实际偏左上** |
| 邮件列表为空 | 没有未领取邮件 | SUCCESS 退出(已领取完) |
| 网络异常 | 领取时断线 | 重新进邮件页 |
| 弹窗阻挡 | 教学弹窗 / 活动弹窗 | dismiss_x 关闭后重试 |
| "一键领取"被遮挡 | 邮件列表在加载中 | 等 1-2s 后重试 |
| 邮件页 X 关闭后仍挡主页 | 弹窗没关 | 再点一次 X |
| 邮件分类 tab 切错 | 选错分类 | 重新点"全部" tab |

## 10. 恢复方式

```
[检测到非主页元素 / 任务卡住]
  ↓
[1. 关闭弹窗]
  - dismiss_x 候选模板扫描 3 次
  - 邮件页 X (1760, 187)
  - 邮件分类 tab 切回"全部"
  ↓
[2. 重试任务]
  - find_mail_entry → tap
  - find_one_key_claim → tap
  - find_close_mail → tap
  ↓
[3. 验证主页]
  - 检测资源栏 + 6 圆盘按钮
  ↓
[4. 主页 → SUCCESS]
  - 3 次循环后仍未到主页 → 报 FAIL
```

## 11. 自动化建议

| 场景 | 推荐技术 | 原因 |
|---|---|---|
| 主页邮件信封 | **多尺度模板匹配**(scale=0.85..1.15) + ROI (30, 380, 130, 120) | 信封较小,需多尺度 |
| 邮件页 X | **多候选模板** + ROI (1690, 120, 130, 90) + tap (1760, 187) | X 实际偏左上 |
| "一键领取" | **多尺度模板匹配** + ROI (530, 880, 280, 105) + tap (670, 932) | 按钮固定位置 |
| 邮件列表加载 | **等待 1-2s** | 列表加载需要时间 |
| 邮件分类 tab | **多候选模板** | 分类可能新增/移除 |
| 邮件未读数 | **不用数字** | 数字变,不能作为模板 |
| 领取结果弹窗 | **dismiss_x 关闭** | 自动消失或点中央 |
